시멘틱(의미있는) 태그

  • 시멘틱 태그란 의미를 담고 있는 태그를 일컫는다
  • header : 홈페이지의 상단영역=>(로고, 제목, 메뉴, 검색, 로그인 SNS)
  • main : 중간영역 =>(홈페이지의 콘텐츠 영역)
  • footer : 하단영역 => (카피라잇, 사이트 이용에 필요한정보, 공지사항, 연락처, 사이트맵)
  • section : 큰 단위의 콘텐츠 그룹 => (콘텐츠 그룹핑, 섹션)
  • article : 작은 단위의 콘텐츠 그룹 => (단독의 하나의 컨텐츠, 쇼핑 섹션 안의 쇼핑몰 하나의 작은단위, 독립적인 컨텐츠, 반복성의 규칙)
  • nav : 홈페이지 메뉴 => (보통 메인메뉴 하나에만 둠)
  • aside : 홈페이지 사이드 영역, 보조컨텐츠 => (날씨정보, 배너광고, 바로가기아이콘, 부가적인 컨텐츠)
  • address : 주소, 주소가 없는 경우에는 카피라잇을 쓰기도 함
  • strong / em : 강조
  • details : 상세보기용 토글버튼
    상세보기 여기는 보일 내용 입력하는 곳

폼문 관련 태그

  • form : 사용자의 정보를 서버로 전송하는 기능의 태그, action과 같이 쓴다
    • 정보 작성




    모름




  • input : 사용자의 정보를 입력받는 태그, type과 같이 씀
  • type.radio : radio 는 반드시 하나만 체크 되어야 함 (강제성)
  • type.checkbox : 자유도
  • filedset : 폼 영역을 알려주는 기능, form 태그의 영역을 알려주는 기능, 보통 같이쓴다
  • legend : 폼영역의 제목, 폼, 필드셋, 리전드 세개를 같이 쓴다 (필드셋 안에 리전드가 반드시 있어야 함)
  • button : 폼 안에 있을 경우 (전송)버튼이다./ 폼 밖에 있을 경우 기능 없는 버튼임
  • label : input의 포커스를 연결해주는 태그 => 글자나 이미지 클릭 시에도 인풋을 받음, label이 감싸고 있지 않은 경우에는 속성에 for를 넣어주고 id로 연결시킨다
  • textarea : 여러 줄 입력
